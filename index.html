PkViz2 — Quick Start Guide


      
What It Does

      
PkViz2 visualizes network packet data as a scatterplot. Each byte in a packet becomes a dot:

      
    
        
  • X axis = byte's position within the packet (0 to 1500)
    • 
        
  • Y axis = byte's value (0 at bottom, 255 at top)
    • 
      


      
How the Visualization Works

      
Packet Window

      
The display shows multiple packets simultaneously. The Window setting controls how many (default: 100). As animation plays, packets slide through: newest enters, oldest drops out.


      
Color = Density

      
When multiple packets share the same position AND value, that pixel shifts from cold (blue) toward hot (yellow). More overlap = warmer color.


      
Brightness = Recency

      
Each packet gets brightness based on age within the window. Newest = full bright, oldest = nearly invisible.


      
Transport Controls (Bottom Bar)

      

        
LoadOpen a .pcap/.pcapng file

        
Load SampleLoad the included sample.pcap

        
ResetJump to first packet

        
« Step / Step »Step one packet back/forward

        
Play / PauseStart/stop animation

        
Go to #Jump to a specific packet number

        
Hex DumpToggle raw hex view of current packet

      


      
Settings (Top Bar)

      

        
WindowPackets displayed at once (default: 100)

        
SpeedPackets per second (default: 10)

        
Max BytesX axis maximum (default: 1500)

        
X ScaleLinear or Logarithmic (spreads early bytes)

        
Invert YFlip Y axis (0 at top)

        
BG / Cold / HotBackground, low-density, and high-density colors

        
Fade Min / MaxBrightness range for oldest/newest packets

      


      
Keyboard Shortcuts

      

        
SpacePlay / Pause

        
Left ArrowStep back

        
Right ArrowStep forward

      


      
Tips

      
    
        
  • Small window (5-10) to see individual packet structure and fade
    • 
        
  • Large window (500+) to see aggregate patterns and density
    • 
        
  • Logarithmic X is great for protocol header patterns
    • 
        
  • DNS traffic (port 53) makes great demo data

PkViz2 — Help


      
Overview

      
PkViz2 visualizes network packet capture files (.pcap / .pcapng) as an animated scatterplot. Every byte in every packet becomes a dot on the canvas:

      
    
        
  • X axis = the byte's position within its packet (0 on the left, up to 1500 on the right)
    • 
        
  • Y axis = the byte's value (0 at the bottom, 255 at the top)
    • 
      

      
Multiple packets are displayed simultaneously in a sliding window. As animation plays, new packets enter the window and old packets drop out, creating a moving view through the capture.


      
How Color and Brightness Work

      
Color = Density

      
Each pixel on the canvas tracks how many packets in the current window have a byte with the same position and same value. When only one packet occupies a pixel, it renders in the cold color (default: blue). As more packets overlap at the same pixel, the color shifts toward the hot color (default: yellow). The scaling is automatic — it adjusts to the densest pixel in each frame, so the full color range is always used.


      
Brightness = Recency

      
Every dot from a single packet shares the same brightness level, determined by the packet's age within the window. The newest packet renders at Fade Max brightness (default: 1.0). The oldest packet renders at Fade Min brightness (default: 0.1), making it nearly invisible against the black background. Packets in between follow a quadratic fade curve, so dots fade quickly at first then hold faintly. Since packets are drawn oldest-to-newest, newer dots always overwrite older ones at shared pixels.


      
Glow Effect

      
A second canvas layer duplicates the image with a soft blur and reduced opacity, giving bright areas a subtle glow or bloom effect. This is purely cosmetic and always active.


      
Loading Data

      

        
LoadOpens a file picker to select a .pcap, .pcapng, or .cap file from your computer. The file is parsed entirely in the browser — nothing is uploaded to a server. Both classic pcap (big/little endian, microsecond/nanosecond) and pcapng formats are supported.

        
Load SampleFetches and loads a sample.pcap file bundled with the application, useful for trying things out without your own capture data.

      


      
Playback Controls

      

        
PlayBegin advancing through packets automatically at the configured Speed. If the animation has reached the end of the capture, pressing Play restarts from the beginning.

        
StopHalt the animation at the current position. The display freezes in place so you can examine the current window.

        
ResetStop playback and jump back to the very first packet in the capture.

        
« StepStop playback and move backward by exactly one packet. The oldest packet in the window re-enters and the newest drops out.

        
Step »Stop playback and move forward by exactly one packet. A new packet enters the window and the oldest drops out.

        
ScrubberThe full-width slider between the canvas and the bottom toolbar. Drag it to quickly jump to any position in the capture. Stops playback when used.

        
Go toType a packet number and click Go (or press Enter) to jump directly to that packet. The display centers the window ending at that packet number.

      


      
Inspection

      

        
Hex DumpToggles a panel below the canvas showing the raw hex and ASCII contents of the current (newest) packet in the window. The display updates automatically as you step or scrub through packets. Format: offset on the left, hex bytes in the center (split into two groups of 8), printable ASCII on the right.

      


      
Status Readout

      

        
PKTThe number of the newest (most recent) packet currently visible in the window.

        
WINDOWThe range of packet numbers currently displayed, e.g. "51-150" means packets 51 through 150 are on screen.

        
TOTALThe total number of packets in the loaded capture file.

      


      
Capture Settings

      

        
WindowHow many packets are visible at once (default: 100). A small window (5–10) lets you see individual packet structures and watch the fade effect. A large window (500+) reveals aggregate patterns and density hot spots across many packets. Range: 1–100,000.

        
SpeedHow many packets advance per second during playback (default: 10). Low speeds let you watch packets arrive one by one. High speeds (100+) create a fast-flowing animation. Range: 0.1–10,000.

        
Max BytesThe maximum byte position shown on the X axis (default: 1500). This sets the right edge of the canvas. Bytes beyond this position in a packet are not displayed. Standard Ethernet MTU is 1500 bytes. Lowering this value zooms into the early bytes of each packet. Range: 1–1500.

      


      
Axes Settings

      

        
X ScaleLinear (default): byte position maps directly to pixel position — each byte gets equal horizontal space. Log: applies a logarithmic mapping so early byte positions (headers, protocol fields) are spread across more horizontal space while later positions (payload) are compressed. This is especially useful for examining packet header structure, where the most meaningful bytes are in the first 20–60 positions.

        
Fill WidthWhen unchecked (default), each byte renders as a single pixel dot. When checked, each byte fills its full horizontal span from its mapped position to the next byte's position. In Linear mode the difference is negligible (each byte already occupies ~1 pixel). In Log mode the effect is dramatic — early bytes that are spread across many pixels render as wide bars instead of isolated dots with gaps between them.

      


      
Palette Settings

      

        
ColdThe color used for low-density pixels, where only one or very few packets share the same byte position and value (default: blue). Click to open a color picker.

        
HotThe color used for high-density pixels, where many packets overlap at the same position and value (default: yellow). The color gradient between cold and hot is computed using a square-root scale, giving more visual range to low-density areas. Click to open a color picker.

      


      
Fade Settings

      

        
MinBrightness of the oldest packet in the window (default: 0.1). At 0 the oldest packet is completely invisible. Higher values keep old packets more visible. Range: 0–10. Values above 1 create an overbright effect.

        
MaxBrightness of the newest packet in the window (default: 1.0). At 1.0 the newest packet renders at full color intensity. Values above 1 push colors toward white for an overbright/bloom look. Range: 0–10.

      


      
Keyboard Shortcuts

      

        
SpaceToggle Play / Stop. Disabled when a text input or dropdown is focused.

        
Left ArrowStep back one packet. Stops playback if running. Disabled when an input is focused.

        
Right ArrowStep forward one packet. Stops playback if running. Disabled when an input is focused.

      


      
Tips

      
    
        
  • Protocol headers create tight horizontal bands at consistent byte values and positions — these are the fixed fields (version, flags, lengths) that every packet shares.
    • 
        
  • Encrypted payloads (TLS, SSH) appear as uniform random noise filling the canvas — a sign that the encryption is working properly.
    • 
        
  • DNS traffic (port 53) makes great demo data: queries are short, responses are variable, and the protocol structure is clearly visible.
    • 
        
  • Try Log X scale with Fill Width enabled to get the best view of header structures.
    • 
        
  • Set Fade Max above 1.0 for an overbright glow on the newest packet.
    • 
        
  • Use the scrubber bar to quickly scan through a large capture, then step through interesting sections one packet at a time.
    • 
        
  • To generate a capture file: sudo tcpdump -i en0 -s 0 -c 1000 -w capture.pcap
